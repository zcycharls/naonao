## Onboarding 引导界面修复

### 问题1：引导时气泡遮挡
- 在 `showMini()` 中加入 `localStorage.getItem('naonao_onboarding_done')!=='true'` 检查
- `finish()` 完成后延迟 800ms 显示问候气泡

### 问题2：引导第2/3步卡片右边直角
- **根因**：`overflow:hidden` + `transform`(scale) 在 Chromium 中裁剪失效
- 尝试过：加 `overflow:hidden auto`、给 `.onboard-input` 加 `box-sizing:border-box` → 均无效
- **最终修复**：`clip-path:inset(0 round 22px)` 强制圆角裁剪 + `.onboard-step.active{overflow:hidden}`
- 验证方法：创建独立 HTML 测试页面对比验证

### 其他 Onboarding 相关
- 设置面板新增"重新引导"按钮（清除 localStorage 标记 + reload）
- `start.sh` 修复：`electron .` → `npx electron .`
- Git SSH 推送修复：移除全局 `url.https://github.com/.insteadof=git@github.com:` 规则
- 调试模式已清理，恢复正常的首次引导逻辑

## 本地 AI 模型管理功能

### 添加"删除本地模型"按钮
- main.js: `local-model:delete` IPC，释放 pipeline + `fs.rmSync` 删除 `userData/models/`
- preload.js: 暴露 `localModelDelete`

- 删除确认弹窗：从原生 `confirm()` 改为 `petDialog.confirm()`（风格统一）

### 删除后状态修复
- 问题1：删除成功后按钮卡在"删除中…" → 在 success 分支恢复按钮文本和 disabled
- 问题2：删除后按钮显示"加载"而非"下载" → `hasDownloadedModel()` 只检查 `userData/`，与 `getModelDir()` 的 dev 回退解耦
- 问题3：下载完成按钮/状态不一致 → 下载成功后直接设置前端 `localModelReady=true; localModelHasFiles=true`，不依赖 IPC 回查

### 下载功能修复
- **HuggingFace 被墙**：设置 `env.remoteHost = 'https://hf-mirror.com'`（已验证 config.json/tokenizer.json/onnx 三个文件 200 OK）
- **进度显示 undefined%**：`progress_callback` 转换 `{status,name,progress}` → `{pct: Math.round(info.progress), msg}`
- **"下载太快"问题**：`@xenova/transformers` 缓存在 `node_modules/.cache/`，删除 `userData/models/` 没清缓存 → `downloadLocalModel()` 设 `env.cacheDir` 到 `userData/models/.cache/`，删除时同时清 `node_modules/@xenova/transformers/.cache/`
