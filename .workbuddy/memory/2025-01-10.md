# 2025-01-10 工作日志

## 完成的工作

### 实现"点击按钮后按需下载本地大模型"功能

**目标：** 将本地 AI 模型从"内置到安装包"改为"点击按钮后按需下载"，大幅减小安装包体积（从 ~722MB 降到 ~260MB）。

**修改的文件：**

1. **package.json** (Task #1 - 已完成)
   - 移除 `extraResources` 配置，阻止模型文件打包到安装包

2. **main.js** (Task #2 - 已完成)
   - 修复 Bug: `hasBuiltInModel()` → `hasDownloadedModel()`
   - 添加 `ipcMain.handle('local-model:download', ...)` IPC 处理器
   - `downloadLocalModel()` 函数已存在，使用 `progress_callback` 报告下载进度
   - 模型存储路径改为 `app.getPath('userData') + '/models'`（用户可写）

3. **preload.js** (Task #3 - 已完成)
   - 添加 `localModelDownload` IPC 桥接
   - 添加 `onLocalModelProgress` 进度监听桥接

4. **app/index.html** (Task #4 - 已完成)
   - 添加下载进度条 UI（默认隐藏）
   - 更新 `getLocalModelStatus()` 函数，处理新状态（未下载、下载中、已下载但未加载等）
   - 更新 `updateStatus()` 函数，正确显示下载按钮和状态
   - 更新 `local-model-btn` 点击事件，支持下载和加载两种流程
   - 添加 `onLocalModelProgress` 监听器，更新下载进度条

**关键技术点：**
- 使用 `@xenova/transformers` v2+ 的 `progress_callback` 监听下载进度
- 使用 HF Mirror (hf-mirror.com) 解决国内访问 Hugging Face 的问题
- 模型存储在 `userData/models/` 目录（确保用户有写权限）
- 进度通过 IPC 从主进程发送到渲染进程

**待完成：**
- 测试下载流程
- 构建新的安装包并验证体积
